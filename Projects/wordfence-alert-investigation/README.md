# Wordfence Alert Investigation

## Overview
This project documents the investigation of a firewall alert showing repeated blocked attacks. The goal was to understand how alerts appear in real environments and how analysts begin basic triage.

## Objective
- Understand firewall alert behavior
- Identify suspicious activity patterns
- Practice basic investigation workflow

## Tools Used
- Wordfence
- AbuseIPDB
- VirusTotal
- WHOIS Lookup

## Steps Performed
1. Reviewed Wordfence alert showing high volume blocked requests
2. Identified repeated inbound traffic patterns
3. Investigated IP reputation using AbuseIPDB
4. Cross-checked with VirusTotal and WHOIS
5. Documented findings and conclusions

## Findings
- High-frequency requests indicate automated scanning behavior
- IP had multiple abuse reports
- Location data does not guarantee attacker identity
